# RP 写作 Checklist

成稿前逐项核对。每项标 ✅/❌,❌ 的必须修掉再交付。

## A. 整体与选题
- [ ] Title 一句话说清做什么,含关键词,无"Novel"等自评词
- [ ] 选题既跟前沿热点又有具体创新角度(非纯重复)
- [ ] 与目标导师近 3 年研究方向有可指出的呼应点
- [ ] 全文有且仅有 1 条主线,所有段落服务于它

## B. 结构完整(按学校要求增减)
- [ ] Cover Page(若要求):姓名/邮箱/学位/导师/题目/日期齐全
- [ ] Abstract:200-300 词,含背景-gap-做法-意义四要素
- [ ] Introduction:漏斗式收敛到 RQ,每个"重要"有证据
- [ ] Literature Review:按主题组织,5C 齐全,收敛到 gap
- [ ] Research Questions:2-3 个,疑问句式,互相有逻辑递进
- [ ] Methodology:占 ~30%,含设计/数据/方法/分析/可行性/风险
- [ ] Timeline:分阶段表格或甘特图,含缓冲
- [ ] Expected Outcomes:理论 + 实践贡献,克制不吹
- [ ] References:另起页,格式统一,真实可核实

## C. 学术质量
- [ ] 每个 RQ 都能在文献综述里找到对应的 gap 支撑
- [ ] 每个方法都解释了"为什么选它而非替代方案"
- [ ] 有明确变量定义(自/因/控制)或问题形式化
- [ ] 有风险评估 + 具体 Plan B(不是"will find alternatives")
- [ ] 创新点明确归类(优化/迁移/结合/scale/解释),不堆"novel"
- [ ] 引用近 3-5 年文献占比 ≥ 50%
- [ ] 无编造引用(拿不准的标 `[需核实]`)

## C-CS. 计算机方向专项(CS RP 必查)
- [ ] RQ 是疑问句,非"用 X 做 Y"的方法陈述
- [ ] baseline ≥ 2-3 个,且为近 1-2 年 SOTA,说明为何选
- [ ] 含消融实验(ablation),逐组件证明贡献
- [ ] 算力写明(GPU 型号/数量/训练时长)+ 开源计划
- [ ] 数据集注明许可与划分,明确无数据泄漏(按 subject/site 划分)
- [ ] 评估指标匹配任务(不平衡用 F1/AUROC;生成用 BLEU/ROUGE+人工;RL 用 return+sample efficiency)
- [ ] 风险预案含"方向被抢发"和"算力不足"两条
- [ ] 多 seed 报均值±std(RL/高方差任务 ≥5 seed)
- [ ] 涉人数据(用户日志/众包/人类评估)有 IRB 说明;LLM 有害性有 red-team 协议

## D. 语言与表达
- [ ] 客观立场,少"I believe/think",多证据推导
- [ ] 无过度承诺("will definitely prove/revolutionize")
- [ ] 术语首次出现给定义或白话解释
- [ ] 长句拆短,每段一个核心意思
- [ ] 无语法/拼写错误(交付前过一遍)
- [ ] 引文动词用对(assert/demonstrate/argue/claim)
- [ ] 占位符用 `<待填:xxx>`,无瞎编的人名/学校/数据

## E. 匹配与可行性
- [ ] RP 与目标院校/导师方向互补(非全问题解决方案)
- [ ] 申请多校时每份 RP 已针对性改写
- [ ] 资源可行性已说明(设备/算力/数据来源)
- [ ] 申请人相关基础有体现(可指向 CV)
- [ ] Timeline 在学位年限内可完成

## F. 格式
- [ ] 字数符合学校要求(常见 1500-3500,部分 5000)
- [ ] 引用格式符合学科规范(IEEE/APA/Harvard/Chicago)
- [ ] 标题层级清晰统一
- [ ] 图表有编号与说明(Timeline 必有)
- [ ] 参考文献不计入正文字数(按学校规定)

## 交付前 3 连问
1. 招生官读完 10 分钟,能一句话说出"这学生要做什么"吗?
2. 导师读完,能判断"这课题接得上我的 lab"吗?
3. 任何一句"重要/合适/创新",都有证据或比较支撑吗?
三个都"是"才交付。
