## 标题

**思维即连续画布：在语言建模中解耦推理、计算与承诺**

（Thought as a Continuous Canvas: Decoupling Reasoning, Compute, and Commitment in Language Modeling）

---

## 摘要

自回归（AR）的下一个 token 预测范式造就了现代语言模型令人瞩目的能力，但它把三件本不必绑定的事情捆绑在了一起：**推理的单元就是输出的单元**、**计算的单元就是 token**、**生成是因果锁定的**——一旦某个 token 被发出就无法再修改，于是多步推理中一个早期错误会沿链路传播。本计划书研究这三重耦合能否被**解耦**，途径是引入一个**连续思维画布**（continuous thought canvas）：一组固定预算的连续向量，作为一个整体在全注意力下被迭代精炼（类似于在一块画板上反复打磨一幅画），再由一个独立的、轻量级的解码器把画布"读"成文字——就像视觉-语言模型从一张固定图片中采样出 caption 那样。

本工作不提出一个完工的系统，而是把一个研究方向结构化，围绕三个问题展开：（RQ1）一块连续画布能否携带足够的"概念级"信息，使一个弱解码器能从中解码出连贯文字，以及画布尺寸如何与输出长度、质量折中；（RQ2）固定迭代次数、固定画布尺寸的生成，能否在产生可变长度输出的同时（一种"类 OCR"性质）得到可预测的计算预算，质量代价几何；（RQ3）置信度门控的迭代精炼——把高置信度向量"承诺"为条件、把低置信度向量扩展——能否减少多步推理中的错误传播。预期影响不是一个榜单提升，而是一套可迁移的、关于"推理 / 计算 / 承诺何时能在神经文本生成中被分离"的理解，以及把大语言模型研究连接到潜在推理、扩散式文本生成与概念级建模的可复用开源产物。

## 1. 引言与背景

Transformer 语言模型逐个、条件地发出一串离散 token。这一自回归（AR）配方造就了当代语言模型的主流能力 [1,2]，其简洁性也是它能 scaling 的重要原因。然而这一设计把三件逻辑上相互独立的事绑在了一起，而绑定的代价正日益显现。

**第一，推理的单元就是输出的单元。** AR 模型无法"想得比说得多"：任何会影响未来的内部计算都必须外化为一个 token，而这个 token 同时也会出现在输出里。链式思维提示 [2] 与 pause token [3] 之类的工作能换取额外计算，但代价仍以"发出的 token"来支付，因此推理预算与输出长度依旧耦合。

**第二，计算的单元就是 token。** 推理成本随输出长度线性增长，且在运行时是数据相关的：同一 prompt 可能产出十个 token 的回答，也可能产出一千个，于是资源预算不可预测。这不止是工程上的不便，而是"模型工作多少"与"模型产出多少"之间的结构性耦合，并由此把一个本质上关于"回答"的过程硬塞进了按 token 计费的模型。

**第三，生成是因果锁定的。** token 一旦发出便无法修改，于是单个早期错误会传播。这是多步推理中被反复记录的失败模式：Transformer 在组合性与多步任务上表现出系统性崩溃，恰因错误会累积 [4]；即便是强模型，没有 self-refine 之类的外部脚手架也几乎无法从一个错误的中间步恢复 [5]。

这三重耦合并非 Transformer 架构本身的必然，而是 **AR token 契约**的必然。本计划书从此处发问：能否在不放弃 Transformer 的前提下替换掉这个契约？具体而言，我们问一个模型能否（i）把推理承载在一种**不是输出**的表示里，（ii）按一种**与输出长度无关的固定预算**花费算力，（iii）在把任何部分"承诺"为输出之前，能修改其部分状态的**任意部分**。

动机来自三条正在汇流的线索。**扩散语言模型** [6,7] 已经通过在全注意力下迭代精炼一个连续表示来生成文本，从原理上证明了"可修改、非因果"的文本生成是可行的——尽管有质量代价。**连续潜在推理** [10]（COCONUT）表明可以训练 Transformer 在连续"思维"向量而非离散 token 上推理，把推理单元与输出 token 解耦。**概念级模型** [11]（LCM）在连续概念空间而非 token 空间建模，提供了"意义可以在 token 之上的表示中被承载与迁移"的证据。每条线索各自解耦了上述三重耦合中的**一重**；没有任何一条同时解耦三重，也没有任何一条把"联合解耦"本身作为研究对象。

本计划书把这一联合解耦确立为一个研究方向。引导性的比喻是：模型先"在脑中画一幅画"——一块固定尺寸的连续画布——再把这幅画"读"成文字，恰如 OCR 或 image-captioning 从一张固定图片中读出文字。画布作为整体被精炼，因此任意向量都能在任意步骤被改变（**可修改**）；画布有固定预算，因此无论最终产出多少文字，计算都是可预测的（**计算可预测**）；画布由一个独立的轻量模块解码，因此推理单元不再是输出单元（**推理解耦**）。这一比喻能否变成一个在真实任务上追平 AR 质量的生成器，是一个尚未定论的实证问题——回答它正是博士阶段要做的事。

## 2. 文献综述

既有工作按四个主题组织。贯穿综述的，是追踪每条线索**解耦了三重耦合（推理/输出、计算/token、因果锁定）中的哪一重**，因为这正是本计划书贡献所在。

**AR 生成及其局限。** AR Transformer 是参考范式 [1]，链式思维提示 [2] 通过发出中间 token 来扩展推理预算。pause-token 训练 [3] 类似地注入可学习的"思考"槽位。两者都**只部分**松开了推理/输出耦合：额外计算仍以（或作为前缀的）已发出 token 来实现，故推理预算与输出长度仍相关，而因果锁定纹丝不动。本计划书针对的 gap 是：尚不存在一种推理表示与输出真正不相交的设定。

**连续与扩散式文本生成。** Diffusion-LM [6] 引入了在词嵌入上的连续扩散，配以退火的类别采样步骤，证明了文本可以由"在全注意力下迭代去噪一个连续表示"来生成（非因果、可修改）。DiffuLLM [7] 与相关连续扩散文本工作 [8] 将其推广到更大规模。Mask-Predict [9] 通过离散的"掩码-预测"循环而非连续去噪实现迭代修改。这些方法**决定性地**松开了**因果锁定**耦合——任意位置都可被修改——但它们仍在 **token 粒度**上操作并把每个位置解码为一个 token，故推理/输出与计算/token 耦合仍在。Gap：可修改的生成已在 token 粒度被研究，但未在"由下游解码的概念/画布"粒度被研究。

**连续潜在推理。** COCONUT [10] 训练 Transformer 用连续"思维"向量替代中间的链式思维 token，把推理单元与输出 token 解耦，并在推理任务上报告了增益。然而 COCONUT 仍**自回归地、顺序地**生成思维——每个思维在下一个产生前就被承诺——因此思维层面因果锁定得以保留，计算仍随已发出思维的数量增长。Gap：连续推理已被引入，但尚未被**可修改化**或**预算固定化**；思维仍是一个一个产出并承诺的。

**概念级与表示空间建模。** 大概念模型（LCM）[11] 在连续概念空间（如多语句向量）而非 token 空间建模序列，提供了"意义可在 token 之上被承载、迁移乃至生成"的证据。这是最接近"概念画布"思想的既有工作，但 LCM 仍**自回归地**生成概念，并用一个现成的编码-解码器把每个概念解码为文字；它既未把画布视为一个可修改、预算固定的整体，也未研究计算可预测性。Gap：概念级生成已被证明可行，但"联合解耦——可修改的概念画板 + 一个单独训练的弱解码器 + 预算固定的精炼"——尚未被作为一个整体来研究。

综上，每条既有线索各自松开一重耦合：扩散/Mask-Predict 打破因果锁定；COCONUT 在思维层面打破推理/输出耦合；LCM 在 token 之上操作。没有一条联合解耦三重，也没有一条把"画布-再-解码"这一实现联合解耦的结构作为研究对象。这正是本计划书所针对的 gap，也是相对既有工作的 concept delta：不是一个更强的生成器，而是对"哪些耦合可被打破、以何种代价打破、所得的可预测性与可修改性是否值得这一代价"的受控研究。

## 3. 研究问题与目标

本计划书围绕三个逻辑递进的问题组织。每个问题都这样表述：其答案**不**由问题设定本身决定——最具信息量的画布尺寸、质量差距的大小、置信度阈值的取值，都是开放的实证问题。

- **RQ1（表示）。** 一块固定预算的连续"思维画布"能否携带足够的"概念级"信息，使一个**单独训练的、刻意做弱**的解码器能从中解码出连贯文字？画布尺寸（连续向量数量）如何与解码出的输出长度、保真度、质量折中？
- **RQ2（解耦计算）。** 固定画布尺寸与精炼迭代次数，能否在产出**可变长度**输出的同时（一种"类 OCR"性质——一张固定图片产出可变文字）得到**可预测的**推理计算预算？在匹配算力下，相对 AR 基线的质量差距几何？
- **RQ3（修改与承诺）。** **置信度门控**的迭代精炼——把超过阈值的向量"承诺"为条件、把低于阈值的向量扩展——能否减少多步推理中的错误传播？置信度阈值如何支配"推理深度 / 质量 / 成本"的折中？

对应目标：

- **O1.** 对"画布-再-解码"生成器，产出"画布尺寸 vs. 解码输出"折中的受控实证刻画。
- **O2.** 量化预算固定生成的"计算可预测性 / 质量"折中，对照 AR 与既有的可修改/连续基线。
- **O3.** 开发并对"置信度门控承诺"作为减少多步推理错误传播的机制做消融。

## 4. 方法与可行性

### 4.1 总体设计

本研究属实证/算法型，采用受控实验结构：基线复现 → 画布-再-解码构建与刻画（RQ1）→ 计算可预测性研究（RQ2）→ 置信度门控精炼（RQ3）→ 消融与鲁棒性检查。本计划书刻意**不**为画布指定单一精炼算子：候选方向包括 (a) 在画布上的去噪扩散（检验"可修改生成是否受益于 score-matching 动力学"的假设）、(b) Mask-Predict 式的迭代填充（一个离散的对照）、(c) 潜在优化 / 摊销推断（检验"画布是否表现为一个变分潜变量"）。在 RQ1 证据基础上在这几者间选择，本身就是一个研究产出，而非一个预先定好的设计。

### 4.2 模型与 checkpoint

所有核心实验使用许可宽松的开源权重 checkpoint：`Qwen/Qwen2.5-1.5B` 与 `Qwen/Qwen2.5-7B` 作为推理/画布模块，`meta-llama/Meta-Llama-3-8B` 作为第二家族做迁移检查；轻量级文本解码器为 0.1B–1B 参数模型，从同家族的 token 级 head 初始化以保证比较公平。精确的 HuggingFace commit hash、`transformers`/`diffusers` 版本与推理引擎（vLLM 版本）将被锁定并记入 Weights & Biases 的 provenance 日志。闭源 API（如 GPT-4 类模型）**仅**用作评估侧的 LLM judge，**绝不**作为核心生成器，从而保证所有主结果可由开源权重复现。训练从这些 checkpoint 出发（继续预训练 / 参数高效微调）；**不提出从零预训练**，与一个现实的学术算力预算一致。

### 4.3 数据

推理数据：GSM8K [12]（MIT 许可）与 MATH [13]（MIT 许可）——多步推理的错误传播正是所关心的失败模式。通用文本：WikiText-103 与 C4 的去重子集（分别为 ODC-BY / ODbL 许可）用于画布-解码保真度研究。划分遵循数据集惯例；为防泄漏，prompt 在各划分间去重，且留出的测试 prompt 绝不进入任何训练或选择阶段（构造上无数据泄漏）。主实验不涉及人类被试；任何人类评估须另行 IRB 修订，已在时间表中预留（约 2–3 个月）。

### 4.4 基线（≥3，近期且对口）

基线的选择跨遍设计空间，使任何发现都不限于单一方法：(i) 标准 **AR Transformer**（Llama-3-8B / Qwen2.5-7B）作参考；(ii) **COCONUT** [10] 作连续潜在推理基线；(iii) **Diffusion-LM / DiffuLLM** [6,7] 作连续扩散文本基线；(iv) **Mask-Predict / CMLM** [9] 作离散可修改基线；(v) **大概念模型** [11] 作概念级基线。有官方 checkpoint 则用之，否则在匹配算力下从同一开源底模重训。

### 4.5 拟议方向（刻意不是一个固定方案）

所研究的生成器有三个松耦合部件，每一个都是一个研究杠杆而非固定设计：一个**画布模块**，在全注意力下维护一组固定尺寸的连续向量；一个**精炼算子**，迭代固定次数（即上述候选算子）；以及一个**轻量解码器**，从最终（或部分承诺的）画布采样出文字，其训练方式如同 image-captioning 从固定图片训练。RQ3 增加一个**置信度头**对每个画布向量打分；高于阈值的向量被"承诺"为下一轮精炼的条件（并可选地解码为文字附在 context 后），低于阈值的向量被扩展（分配额外画布维度）——类似于把温度不断降低、趋近低能最终态。承诺阈值直接控制推理深度，给出一个关于"推理/质量/成本"折中的单一旋钮。本计划书不固定阈值调度、精炼算子或解码器容量；这些都在 RQ1–RQ2 证据基础上选定。

### 4.6 评估与统计

**质量（生成）**：通用文本上 ROUGE-L 与 BLEU，加一个冻结参考 LM 下的困惑度。**质量（推理）**：GSM8K/MATH 上的精确匹配准确率与 pass@$k$，以及一个**错误注入**协议——强制一个错误中间步并测量恢复——直接探查"可修改"这一主张。**计算可预测性**：推理 FLOPs/延迟对输出长度的方差，比较 AR（数据相关）与预算固定的画布生成；"类 OCR"性质被操作化为"计算方差与输出长度方差之比"。**推理深度**：准确率 vs. 承诺阈值、vs. 画布尺寸。所有实验用 ≥3 个随机种子（推理与扩散训练属中高方差），报告均值±标准差；对准确率与胜率做配对 bootstrap 显著性检验。消融逐个移除部件：画布尺寸、解码器容量、精炼迭代数、置信度头。

### 4.7 算力与可复现

硬件：4×A100 80GB，可选经院校 HPC 集群扩展到 8×A100。预估成本：一次 7B 的继续预训练 / LoRA 在画布模块上约 50–150 GPU 小时；RQ1–RQ3 在基线 × 画布尺寸 × 种子上的完整刻画，估约 1.5–2.5k GPU 小时（两年内），在预算内；任何 8B+ 运行用 QLoRA。所有代码、配置、prompt（版本化）、种子、所选数据索引与 provenance 日志将以 GitHub + Zenodo DOI 发布；Docker 镜像锁定依赖以做 bit 级复现。QLoRA/GPTQ-AWQ 运行将报告多种子方差，因量化推理路径已知存在非确定性。

### 4.8 可行性与风险预案

| 风险 | 概率 | 影响 | 预案 |
|---|---|---|---|
| 方向被抢发（arXiv；潜在推理是热点） | 中 | 高 | 每月 arXiv 监测；以"联合解耦"框架与"置信度门控承诺"研究做差异化；预注册的备选 RQ——跨算子迁移。 |
| 相对 AR 的质量差距过大、信息量不足 | 中 | 高 | 先在 GSM8K/MATH 上做，推理增益可对冲生成质量损失；报告"质量/计算"帕累托前沿而非单一数字；以更强解码器作诊断上界。 |
| 连续扩散文本训练不稳定 | 中 | 中 | 借用 Diffusion-LM 的退火调度；退而用 Mask-Predict 算子（离散、稳定）作受控对照。 |
| 轻量解码器成为瓶颈 | 中 | 中 | 把解码器容量作为消融轴；通过对画布向量的 probing 诊断信息损失。 |
| 算力不足 / 排队 | 中 | 高 | 8B+ 用 QLoRA；先在 1.5B/7B 上小规模验证；Y1 申请 HPC 配额；租云 A100 兜底。 |

## 6. 预期成果与意义

**理论贡献。** 一套关于"推理、计算与承诺何时能在神经文本生成中被分离"的可迁移刻画——不是一个新 SOTA 生成器，而是对"哪些耦合可被打破、以何种代价打破、所得可预测性/可修改性是否值得这一代价"的回答。这回答了超越榜单位移的"so what"：预期影响是对 AR-token 契约及其替代物的可复用理解，而非 2% 的基准提升。

**实践贡献。** 开源的画布-解码实现、精心设计的评估协议（含探查可修改性的错误注入测试）与计算可预测性基准，使"可修改、预算固定"的生成对仅有几张 A100 的学术实验室可达。单一阈值的推理深度旋钮（承诺阈值）是一个具体、可部署的产物。

**更长期方向（作为动机陈述，非承诺成果）。** 由于画布是一个连续的、模态无关、可替换解码头的表示，同一结构在**原理上**是连接语言生成与潜在推理、世界模型、视觉-语言-动作（VLA）设定的一条路径。这些连接作为"为何解耦值得研究"的动机陈述；它们明确**不**是本博士的承诺成果，本计划书范围严格限定在上述文本与推理设定内。

**与目标课题组的匹配。** 本方向与导师在高效与结构化语言建模方面的工作互补 `<待填：引用导师近 3 年 2–3 篇论文，并说明可复用课题组的开源代码>`；它产出可复用产物（画布-解码模块、错误注入基准、计算可预测性曲线），可为潜在推理与可修改生成的后续工作埋下种子。"解耦 AR 契约"而非"再提一个 AR 变体"的互补定位，最大化协同而非重叠。
